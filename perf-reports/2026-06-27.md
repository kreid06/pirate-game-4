## Applied Changes

### Change 1: Right-size shared blob JSON buffers + truncation guards

**File:** `server/src/net/websocket_server.c`  
**Struct:** `SharedBlobOutput`

**What:**
- `co_json`: 1024 → 8192 B (64 companies × ~85 B ≈ 5.4 KB worst case)
- `tmb_json`: 4096 → 8192 B (64 tombstones × ~120 B with `ownerName[64]` ≈ 7.7 KB)
- `projectiles_json`: 32768 → 65536 B (500 projectiles × ~95 B ≈ 47 KB)
- Added headroom checks and `(size_t)` remaining-space guards on tombstone, company, and projectile append loops (matching existing `ditem_json` pattern)

**Evidence:** Buffer math shows prior caps could overflow at max entity counts; unsigned `sizeof - int offset` could wrap once offset exceeded cap. Truncation now breaks cleanly with warn log on projectiles.

**Risk:** None. Larger static worker buffers only; semantics unchanged at typical loads.

---

### Change 2: Double-buffer blob worker job snapshot (skip struct copy)

**File:** `server/src/net/websocket_server.c`  
**Functions:** `blob_worker_main`, `blob_worker_submit_snapshot`

**What:** Replaced single `SharedBlobSnapshot job` + value copy (`local_job = g_blob_worker.job`) with `job_bufs[2]` double-buffer. Main thread fills `job_bufs[job_write_idx]`, worker reads `job_bufs[job_read_idx]` by pointer.

**Evidence:** `SharedBlobSnapshot` holds `SimpleShip ships[200]`, `struct Ship sim_ships[200]`, `BlobPlayer players[100]`, etc. — multi-MB per tick copied even though submit already uses count-guarded memcpy into active slots only. Worker now reads in-place from the inactive buffer (mirrors existing `output_bufs[2]` pattern).

**Risk:** None. Mutex/cond handoff unchanged; write/read indices flip atomically under lock.

---

### Change 3: Send loop iterates compact active slot lists

**File:** `server/src/net/websocket_server.c`  
**Function:** `websocket_server_send_game_state` (phase 1)

**What:** Player AOI assembly iterates `player_active_slots[]` (count `player_active_slot_count`) instead of scanning all `WS_MAX_CLIENTS` (100). NPC AOI assembly iterates `npc_active_slots[]` instead of `0..npc_entry_count`.

**Evidence:** Blob build already maintains these compact lists when marking entries active. With 0 online players and ~80–150 NPCs, prior send loop still scanned 100 player slots and up to 512 NPC index slots per connected client per tick.

**Risk:** None. Same entries emitted; fewer branch checks on idle slots.

---

### Change 4: Binary search for sorted sim entity lookups

**File:** `server/src/sim/simulation.c`  
**Functions:** `sim_get_ship`, `sim_get_player`, `sim_get_projectile`

**What:** Replaced O(n) linear scans with O(log n) binary search. Ship/player/projectile arrays are kept sorted by ID each tick via insertion sort (already documented in `sim_update_ships` / player / projectile update paths).

**Evidence:** At max entities (200 ships, 100 players, 500 projectiles), hot-path lookups from input processing, collisions, and snapshot generation previously scanned up to full array length per call.

**Risk:** None. Sort invariant already maintained; lookup semantics identical.

---

### Change 5: Reuse validated player pointer in `sim_process_input`

**File:** `server/src/sim/simulation.c`  
**Function:** `sim_process_input`

**What:** Removed redundant `sim_get_player(sim, cmd->player_id)` calls in `PLAYER_ACTION_FIRE_CANNON` and `PLAYER_ACTION_JUMP` handlers; reuse the pointer validated at function entry.

**Evidence:** Two extra linear (now binary) lookups per action command on an already-resolved player.

**Risk:** None.

---

### Change 6: `format_dominators_extra` closing bracket size cast

**File:** `server/src/net/websocket_server.c`  
**Function:** `format_dominators_extra`

**What:** `snprintf(buf + n, cap - n, "]")` → `snprintf(buf + n, (size_t)(cap - n), "]")` to match the per-element writes on line 56.

**Evidence:** Signed `cap - n` passed where `size_t` is expected; defensive consistency with prior comma-fix audit.

**Risk:** None.

## Proposed (Needs Human Review)

### A. `per_gs_pool` -Warray-bounds: oversized payload silent truncation

When `_goff + len` exceeds `PER_GS_BUF`, payloads are silently truncated. Fix needs full `size_t _goff` hard-cap audit across the send assembly loop.

### B. Prebuild shared proj/tmb/ditem/co JSON tail once per tick

Per-client GAME_STATE assembly re-memcopies identical projectile/tombstone/ditem/company sections for every connected client. High impact at 20+ clients; requires careful JSON field-order preservation.

### C. Ship JSON dirty-flag cache (player/NPC pattern)

Per-ship `snprintf` rebuild runs every tick even when sim state unchanged. Needs stale-field audit (modules, claim flags, mass) before skipping.

### D. `sim_update_spatial_hash` full 1.5 MB memset every tick

Incremental dirty-cell clearing needs collision correctness audit.

### E. Grapple-target player dirty key extension

`player_is_grapple_target()` forces full player reserialize every tick; anchor coords could be folded into `player_key`.

### F. UDP `snapshot.c` delta baseline O(n²) entity lookup

UDP snapshot path not active at runtime; baseline LUT would help if re-enabled.
