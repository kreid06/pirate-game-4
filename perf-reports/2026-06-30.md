## Applied Changes

### Change 1: Skip tombstone/dropped-item blob copies when nothing is live

**File:** `server/src/net/websocket_server.c`  
**Functions:** `copy_tombstones_to_blob`, `copy_dropped_items_to_blob`

**What:** Return `0` immediately when `tombstone_live_count == 0` or `dropped_item_live_count == 0`, skipping 64- and 256-slot scans on every blob-worker submit.

**Evidence:** JSON builders already fast-path on `snap->tombstone_active_count == 0` / `dropped_item_active_count == 0`. The copy helpers ran first each tick regardless. On idle ghost-fleet servers (no deaths, no manual drops) this was 320 slot visits/tick on the main thread plus worker input path.

**Risk:** None. Downstream only reads tombstone/ditem slots when active counts are non-zero; `active_count == 0` selects the `[]` JSON fast path.

---

### Change 2: Shallow-water drag island broad-phase

**File:** `server/src/sim/simulation.c`  
**Function:** `sim_update_ships` (shallow-water block)

**What:**
- Skip polygon islands with `shallow_vertex_count == 0` (exact — `island_shallow_water_depth` returns `0.0f`).
- For bump islands, skip when ship centre is outside `broad_outer = beach_radius_px + beach_max_bump + shallow_depth` (same conservative radius used inside `island_shallow_water_depth`).

**Evidence:** Prior code called `island_shallow_water_depth` for all 9 islands on every ship every tick (~150 ships × 9 calls). Open-ocean ghost ships still paid polygon vertex loops and bump-island `atan2f`/`island_boundary_r` work even when hundreds of pixels from any shallow zone.

**Risk:** None. Filters are conservative (same math as the inline helper's own early returns); shallow drag coefficients unchanged when inside a zone.

---

### Change 3: Early-out collision handlers when ship count is too low

**File:** `server/src/sim/simulation.c`  
**Functions:** `handle_island_collisions`, ship-vs-ship CCD block in `sim_handle_collisions`, `handle_player_ship_collisions`

**What:**
- `handle_island_collisions`: return when `sim->ship_count == 0`.
- Ship-vs-ship CCD: run only when `sim->ship_count >= 2` (mirrors `handle_ship_collisions`).
- `handle_player_ship_collisions`: also return when `sim->ship_count == 0`.

**Evidence:** Island collision outer loop always iterated `ISLAND_COUNT` even with zero ships. CCD ship loop ran with `ship_count == 1` (inner `j` loop empty). Player-ship handler already guarded `player_count == 0` but not `ship_count == 0`.

**Risk:** None. Guards only skip loops that had zero iterations anyway.

---

### Change 4: Direct player JSON write into blob cache

**File:** `server/src/net/websocket_server.c`  
**Function:** `build_shared_blobs_from_snapshot` (player entry block)

**What:** Write player JSON directly into `out->player_entry[p]` instead of a local `char player_entry[3200]` stack buffer followed by `memcpy`.

**Evidence:** NPC entries already use direct `snprintf(out->npc_entry[n], …)`. Moving players fail the dirty check every tick, so the stack buffer + memcpy ran at ~30 Hz per active moving player (~3.2 KB copy each).

**Risk:** None. Same bounds checks and grapple suffix patching; mirrors existing NPC pattern.

## Proposed (Needs Human Review)

### A. `per_gs_pool` -Warray-bounds: oversized payload silent truncation

When `_goff + len` exceeds `PER_GS_BUF`, payloads are silently truncated. Fix needs full `size_t _goff` hard-cap audit across the send assembly loop.

### B. Prebuild shared proj/tmb/ditem/co JSON tail once per tick

Per-client GAME_STATE assembly re-memcopies identical projectile/tombstone/ditem/company sections for every connected client. High impact at 20+ clients; requires careful JSON field-order preservation.

### C. Ship JSON dirty-flag cache (player/NPC pattern)

Per-ship `snprintf` rebuild runs every tick even when sim state unchanged. Needs stale-field audit (modules, claim flags, mass) before skipping.

### D. `sim_update_spatial_hash` full 1.6 MB memset every tick

Spatial hash is built but not consumed by collision code today. Removing or incrementally clearing needs collision correctness audit before wiring a reader.

### E. Grapple-target player dirty key + one-pass grapple index

`player_is_grapple_target()` scans all 100 grapple slots per player dirty check; dirty path scans again for suffix patch. Precompute a target table once per blob build.

### F. Global player active slot list on blob submit

Output path uses compact active lists; submit path still scans all `WS_MAX_CLIENTS` slots each tick.

### G. Skip blob-worker submit when zero websocket clients connected

Would save ~1 ms/tick on idle ghost-fleet servers but needs a reliable connected-client counter on every connect/disconnect path.

### H. UDP `snapshot.c` delta baseline O(n²) entity lookup

UDP snapshot path not active at runtime; baseline id→index LUT would help if re-enabled.
