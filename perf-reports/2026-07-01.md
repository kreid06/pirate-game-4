## Applied Changes

### Change 1: Early-out `sim_update_players` when no sim players

**File:** `server/src/sim/simulation.c`  
**Function:** `sim_update_players`

**What:** Return immediately when `sim->player_count == 0`, before insertion sort and physics loop setup.

**Evidence:** Ghost-fleet profiling runs have `sim->player_count == 0` yet the function still ran insertion sort setup every 30 Hz tick. Mirrors existing guards in `handle_player_ship_collisions` and `handle_player_boulder_collisions`.

**Risk:** None. When count is zero the loops had zero iterations anyway.

---

### Change 2: Early-out `sim_update_projectiles` when no projectiles

**File:** `server/src/sim/simulation.c`  
**Function:** `sim_update_projectiles`

**What:** Return immediately when `sim->projectile_count == 0`, before insertion sort and the per-projectile island over-land scan loop.

**Evidence:** Idle servers with no active cannon fire still paid sort setup plus up to `projectile_count × ISLAND_COUNT` island checks when count was zero (loop guard prevented inner work but not function entry). Mirrors `handle_projectile_collisions` early-out from 2026-06-29.

**Risk:** None.

---

### Change 3: `simulation_has_entity` via binary search

**File:** `server/src/sim/simulation.c`  
**Function:** `simulation_has_entity`

**What:** Replace three linear scans over player/ship/projectile arrays with existing `sim_get_player`, `sim_get_ship`, and `sim_get_projectile` (O(log n) binary search on sorted-by-ID arrays).

**Evidence:** Called from UDP `network_send_snapshots` before each snapshot send. At max entity counts the prior path did up to 800 comparisons per call; binary search is O(log n) per type with identical lookup semantics.

**Risk:** None.

---

### Change 4: Narrow snapshot packet zero-init scope

**File:** `server/src/net/snapshot.c`  
**Function:** `snapshot_generate_for_player`

**What:** Replace `memset(packet, 0, sizeof(struct SnapshotPacket))` with zeroing only `packet->header` and setting `entity_count = 0`.

**Evidence:** `sizeof(struct SnapshotPacket)` includes a 64-slot entity union (~896 B) plus header, but wire size is computed from actual `entity_count` (baseline) or delta byte sum (delta path). Full struct zero cleared ~900 B per UDP snapshot generation call unnecessarily. Baseline path writes every emitted slot; delta path writes only `delta_count` entries and computes size from delta lengths.

**Risk:** None. Unwritten union tail bytes are never serialized.

---

### Change 5: `find_sim_ship` cache miss uses binary search

**File:** `server/src/net/websocket_server.c`  
**Function:** `find_sim_ship`

**What:** On LUT cache miss, call `sim_get_ship(global_sim, id)` instead of linear scan over `global_sim->ship_count` (up to 200). Still populate `g_sim_ship_by_id` on hit.

**Evidence:** 28 call sites use `find_sim_ship`; `sync_simple_ships_from_simulation` rebuilds the touched LUT each tick but misses still occurred for uncached IDs. `sim_get_ship` is O(log n) with the sorted-by-ID invariant maintained each tick.

**Risk:** None.

## Proposed (Needs Human Review)

### A. `per_gs_pool` -Warray-bounds: oversized payload silent truncation

When `_goff + len` exceeds `PER_GS_BUF`, payloads are silently truncated. Fix needs full `size_t _goff` hard-cap audit across the send assembly loop.

### B. Prebuild shared proj/tmb/ditem/co JSON tail once per tick

Per-client GAME_STATE assembly re-memcopies identical projectile/tombstone/ditem/company sections for every connected client. High impact at 20+ clients; requires careful JSON field-order preservation.

### C. Ship JSON dirty-flag cache (player/NPC pattern)

Per-ship `snprintf` rebuild runs every tick even when sim state unchanged. Needs stale-field audit (modules, claim flags, mass) before skipping.

### D. `sim_update_spatial_hash` full 1.6 MB memset every tick

Spatial hash is built but not consumed by collision code today. Removing or incrementally clearing needs collision correctness audit before wiring a reader.

### E. Global player active slot list on blob submit

Output path uses compact active lists; submit path still scans all `WS_MAX_CLIENTS` slots each tick.

### F. Skip blob-worker submit when zero websocket clients connected

Would save ~1 ms/tick on idle ghost-fleet servers but needs a reliable connected-client counter on every connect/disconnect path.

### G. UDP `snapshot.c` delta baseline O(n²) entity lookup

UDP snapshot path not active at runtime; baseline id→index LUT would help if re-enabled.

### H. Snapshot baseline direct quantize into packed union

Writing directly into `packet->baseline_entities[]` avoids temp+memcpy but triggers new `-Waddress-of-packed-member` warnings; needs struct layout change or pragma policy decision.

### I. Unmerged `perf/daily-2026-06-30` changes still on develop

That branch (not yet merged) contains additional safe wins: tombstone/ditem copy early-outs, shallow-water island broad-phase, island/CCD/player-ship ship_count guards, and direct player JSON write. Consider merging that PR before the next daily run to avoid re-discovering the same optimizations.
