## Applied Changes

### Change 1: Compact player/NPC active slot lists in blob snapshot

**File:** `server/src/net/websocket_server.c`  
**Struct:** `SharedBlobSnapshot`  
**Functions:** `blob_worker_submit_snapshot`, first-tick fallback snapshot, `build_shared_blobs_from_snapshot`

**What:**
- Added `player_active_slots[]` / `player_active_slot_count` and `npc_active_slots[]` / `npc_active_slot_count` to `SharedBlobSnapshot`.
- Populate these compact lists when copying players/NPCs into the worker job (and first-tick fallback).
- Blob build loops iterate only active slots instead of scanning all `WS_MAX_CLIENTS` (100) player indices or all `world_npc_count` NPC index slots (often 80–150 with sparse inactive entries).

**Evidence:** With 0 online players and ~80–150 active NPCs, the prior NPC blob path scanned every index `0..world_npc_count-1` and branched on `npc->active` for each. Player path scanned 100 slots even when 0 players were online. The output side already used compact `player_active_slots` / `npc_active_slots`; this mirrors that pattern on the snapshot input side.

**Risk:** None. Same entries serialized; fewer branch checks on inactive indices.

---

### Change 2: `(size_t)` remaining-space guards on ship JSON module snprintf

**File:** `server/src/net/websocket_server.c`  
**Function:** `build_ships_blob_from_snapshot` (sim-ship and simple-ship paths)

**What:** All 19 `snprintf(ship_entry + offset, sizeof(ship_entry) - offset, …)` calls now use `(size_t)sizeof(ship_entry) - (size_t)offset`.

**Evidence:** Signed `sizeof - int offset` wraps to a huge `size_t` once `offset` exceeds `sizeof(ship_entry)` (16384), matching the documented overflow class that corrupted adjacent blob struct fields in the NPC combined-buffer bug (2026-06-27 report). Module append loops already guard `offset < sizeof - 200`; the cast closes the snprintf size argument path.

**Risk:** None. Defensive bounds consistency only.

---

### Change 3: `(size_t)` remaining-space guards on player inventory JSON snprintf

**File:** `server/src/net/websocket_server.c`  
**Function:** `build_shared_blobs_from_snapshot` (player dirty rebuild)

**What:** Inventory `inv_buf` append snprintf calls use `(size_t)(sizeof(inv_buf) - (size_t)inv_off)`.

**Evidence:** Same signed/unsigned wrap class as Change 2; inventory JSON is built only on dirty player ticks but uses the same anti-pattern.

**Risk:** None.

### Change 4: Cache firing-ship pointer per projectile in collision handler

**File:** `server/src/sim/simulation.c`  
**Function:** `handle_projectile_collisions`

**What:** Resolve `firing_ship = sim_get_ship(…)` once per projectile iteration; reuse for ghost-immunity checks and XP award paths instead of up to 8 repeated binary searches per projectile × ship pair.

**Evidence:** Prior path called `sim_get_ship` inside the per-ship inner loop for ghost immunity and again at each XP award site (7 additional call sites). With 500 projectiles and 150+ ships this is thousands of redundant O(log n) lookups per tick during fleet fights.

**Risk:** None. Pointer resolved once per projectile before the ship loop; semantics unchanged.

## Proposed (Needs Human Review)

### A. `per_gs_pool` -Warray-bounds: oversized payload silent truncation

When `_goff + len` exceeds `PER_GS_BUF`, payloads are silently truncated. Fix needs full `size_t _goff` hard-cap audit across the send assembly loop.

### B. Prebuild shared proj/tmb/ditem/co JSON tail once per tick

Per-client GAME_STATE assembly re-memcopies identical projectile/tombstone/ditem/company sections for every connected client. High impact at 20+ clients; requires careful JSON field-order preservation.

### C. Ship JSON dirty-flag cache (player/NPC pattern)

Per-ship `snprintf` rebuild runs every tick even when sim state unchanged. Needs stale-field audit (modules, claim flags, mass) before skipping.

### D. `sim_update_spatial_hash` full 1.5 MB memset every tick

Incremental dirty-cell clearing needs collision correctness audit.

### E. Grapple-target player dirty key extension

`player_is_grapple_target()` forces full player reserialize every tick; anchor coords could be folded into `player_key`.

### F. UDP `snapshot.c` delta baseline O(n²) entity lookup

UDP snapshot path not active at runtime; baseline id→index LUT would help if re-enabled.
