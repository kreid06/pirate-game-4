## Applied Changes

### Change 1: `memcpy` instead of `snprintf("%s")` for ship JSON append

**File:** `server/src/net/websocket_server.c`  
**Function:** `build_ships_blob_from_snapshot` (sim-ship and simple-ship paths)

**What:** After building each `ship_entry[]`, append to `ships_json` via `memcpy(…, offset)` instead of `snprintf("%s", ship_entry)`.

**Evidence:** Every active ship previously paid a libc format-string dispatch to copy an already-formatted C string. With 150+ ghost ships at 5–30 Hz this is 750–4500 redundant snprintf calls/s. `offset` is the known byte length; bounds use `size_t` room check (same cap semantics as prior snprintf guard).

**Risk:** None. Bytes copied and bounds identical to prior `%s` append.

---

### Change 2: Skip zero-count `memset` on blob snapshot companies / claim flags

**File:** `server/src/net/websocket_server.c`  
**Functions:** `blob_worker_submit_snapshot`, first-tick fallback snapshot assembly

**What:** Removed `memset` of full `dynamic_companies[]` (~2.7 KB) and `claim_flags[]` when respective counts are zero. JSON builders and claim LUT already gate on `dynamic_company_count` / `claim_flag_count`.

**Evidence:** Idle / fresh worlds have 0 companies and 0 claim flags; prior path cleared ~2.7 KB + 16×sizeof(ClaimFlag) on every 5–30 Hz blob submit despite count-guarded consumers never reading stale slots.

**Risk:** None. When count drops from N→0, consumers iterate 0 entries; stale tail slots are unreachable.

---

### Change 3: Conditional inactive-player clear in blob submit

**File:** `server/src/net/websocket_server.c`  
**Function:** `blob_worker_submit_snapshot`

**What:** Only write `job.players[slot].active = false` when that slot was active on the previous build (`else if (g_blob_worker.job.players[_bpi].active)`).

**Evidence:** With 0 online players (common idle case), prior path wrote 100 boolean clears every tick even though `g_blob_worker` is BSS-zeroed and slots stay false after the one post-disconnect clear.

**Risk:** None. `g_blob_worker` is zero-initialized; first disconnect still clears via the conditional branch.

---

### Change 4: Early-exit on live-count in tombstone / dropped-item copy and JSON loops

**File:** `server/src/net/websocket_server.c`  
**Functions:** `copy_tombstones_to_blob`, `copy_dropped_items_to_blob`, `build_shared_blobs_from_snapshot`

**What:** Break snapshot-copy loops once `active >= tombstone_live_count` / `dropped_item_live_count`. Break JSON serialization loops once processed count reaches `snap->tombstone_active_count` / `snap->dropped_item_active_count`.

**Evidence:** With sparse active items (typical: 0–3), prior paths scanned all 64 tombstone or 256 dropped-item slots even after finding every live entry. Same pattern as 2026-06-24 zero-active fast path, extended to the non-zero sparse case.

**Risk:** None. Counters maintained at spawn/collect/expiry sites; loop still checks `.active` before counting.

---

### Change 5: Direct comma write in `format_dominators_extra`

**File:** `server/src/net/websocket_server.c`  
**Function:** `format_dominators_extra`

**What:** Replaced per-element `snprintf("%s%u", i ? "," : "", …)` with `buf[n++] = ','` plus `snprintf("%u", …)` using `(size_t)(cap - n)` for remaining space.

**Evidence:** Same class of win as 2026-06-15 / 2026-06-25 comma fixes. Structures with dominator lists are rare but the helper is shared by structure JSON formatters.

**Risk:** None. Bounds guard preserved.

## Proposed (Needs Human Review)

### A. `per_gs_pool` -Warray-bounds: oversized payload silent truncation

When `_goff + len` exceeds `PER_GS_BUF`, payloads are silently truncated. Fix needs full `size_t _goff` hard-cap audit across the send assembly loop.

### B. Ship JSON dirty-flag cache (player/NPC pattern)

Per-ship `snprintf` rebuild runs every tick even when sim state unchanged. Needs stale-field audit (modules, claim flags, mass) before skipping.

### C. `sim_update_spatial_hash` full 1.5 MB memset every tick

Incremental dirty-cell clearing needs collision correctness audit.

### D. `placed_structure_count` linear scans in movement hot path

Structure-ID hash would help but touches data layout.

### E. UDP `snapshot.c` delta baseline O(n²) entity lookup

UDP snapshot path not active at runtime; deferred.

### F. Signed/unsigned `sizeof - int offset` in remaining ship_entry module snprintf bounds

Several module loops still use `(int)sizeof(ship_entry) - offset`; audit for wrap once offset exceeds buffer size (same root cause as the 2026-06-23 npcs_json overflow fix).
