## Applied Changes

### Change 1: Touched-slot clearing for `player_entry_active` / `npc_entry_active`

**File:** `server/src/net/websocket_server.c`  
**Function:** `build_shared_blobs_from_snapshot`

**What:** Replaced full `memset` of 100 + 512 boolean slots every blob build with per-buffer touched-slot lists (`player_active_slots[]`, `npc_active_slots[]` in `SharedBlobOutput`). Only indices marked active on the previous build are cleared before re-marking current actives.

**Evidence:** 612 bytes cleared every 5–30 Hz blob build even when only ~18 NPC slots and 0 player slots are live. Double-buffered `SharedBlobOutput` keeps separate touched lists per buffer, preserving correctness when read/write indices flip.

**Risk:** None. Despawned entities are cleared via the previous tick's touched list; semantics match full memset.

---

### Change 2: Static touched claim-flag lookup table in ship JSON builder

**File:** `server/src/net/websocket_server.c`  
**Function:** `build_ships_blob_from_snapshot`

**What:** Replaced stack `ClaimFlag* claim_by_ship_id[SIM_SHIP_ID_SIZE] = {0}` (512 pointers, 4 KB zero-init every build) with BSS `g_claim_by_ship_id[]` plus `claim_lut_reset()` / `claim_lut_set()` touched invalidation (≤ `MAX_CLAIM_FLAGS = 16` entries).

**Evidence:** Same pattern as `sim_ship_lut` (2026-06-24) and `SnapShipLut` (2026-06-19). When `claim_flag_count == 0` the reset is a no-op; when flags are active only touched ship IDs are cleared/repopulated.

**Risk:** None. Lookup semantics unchanged; worker thread is single-threaded for this path.

---

### Change 3: Direct comma writes in ship module JSON loops

**File:** `server/src/net/websocket_server.c`  
**Functions:** `build_ships_blob_from_snapshot` (sim + simple ship paths), handshake initial-state ship builder

**What:** Replaced up to ~64 × ships `snprintf("%s", m > 0 ? "," : "")` module separators with a single `ship_entry[offset++] = ','` (or `ships_str[ships_offset++] = ','`) at the top of each module loop, removing the `%s` comma argument from all module format strings.

**Evidence:** Same class of win as 2026-06-15 projectile/ship-array comma fix. Brigantine ships have ~40–64 modules; at 30 Hz ship JSON rebuilds this avoids hundreds of libc format-string dispatches per second under combat load.

**Risk:** None. Bounds guard (`offset < cap - 1`) matches prior snprintf guard.

## Proposed (Needs Human Review)

### A. `per_gs_pool` -Warray-bounds: oversized payload truncation

When `_goff + len` exceeds `PER_GS_BUF`, payloads are silently truncated. Fix needs `size_t _goff` hard cap audit across the send assembly loop.

### B. `sim_update_spatial_hash` full 1.5 MB memset every tick

Incremental dirty-cell clearing needs collision correctness audit.

### C. `placed_structure_count` linear scans in movement hot path

Structure-ID hash would help but touches data layout.

### D. UDP `snapshot.c` delta baseline O(n²) entity lookup

UDP snapshot path not active at runtime; deferred.

### E. Signed/unsigned `sizeof - int offset` in remaining snprintf bounds

Several send-path and blob-path calls still use `(int)sizeof(buf) - offset`; audit for wrap once offset exceeds buffer size (same root cause as the 2026-06-23 npcs_json overflow fix).
