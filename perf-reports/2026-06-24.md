## Applied Changes

### Change 1: Double-buffer `SharedBlobOutput` ‚Äî eliminate struct copies

**File:** `server/src/net/websocket_server.c`  
**Functions:** `blob_worker_main`, `blob_worker_peek_output`, `websocket_server_send_game_state`

**What:** Replaced worker stack `local_out` + mutex struct assignment + main-thread `*out = g_blob_worker.output` copy with two BSS `output_bufs[2]`. Worker builds into the write buffer and publishes by flipping `output_read_idx`; main thread reads via `const SharedBlobOutput*` without copying.

**Evidence:** `SharedBlobOutput` is ~2.7 MB (includes 2 MB `ships_json`, 100√ó3.2 KB `player_entry`, etc.). Two full copies per tick at 30 Hz ‚âà 162 MB/s memcpy bandwidth on top of JSON build work. Post-change `ships=` last-build dropped from 339‚Äì546 ¬µs to 179‚Äì265 ¬µs on idle server.

**Risk:** None. Mutex ordering unchanged; stale-frame fallback preserved.

---

### Change 2: Zero-active tombstone/dropped-item snapshot copy fast path

**File:** `server/src/net/websocket_server.c`  
**Functions:** `copy_tombstones_to_blob`, `copy_dropped_items_to_blob`, spawn/collect/expiry paths

**What:** Added `tombstone_live_count` / `dropped_item_live_count` maintained at spawn, collect, pickup, grapple, and expiry sites. When count is zero, snapshot copy returns immediately without scanning 64 + 256 slots every tick.

**Evidence:** Idle server has 0 tombstones and 0 dropped items; prior path scanned 320 slots on every 5‚Äì30 Hz blob submit even though JSON builder already fast-pathed empty arrays (2026-06-23).

**Risk:** None. Counters mirror existing `claim_flag_count` pattern; helpers clamp decrement to avoid underflow.

### Change 3: Touched-slot invalidation for `g_sim_ship_by_id`

**File:** `server/src/net/websocket_server.c` ‚Äî `sync_simple_ships_from_simulation`

**What:** Replaced full 4 KB `memset(g_sim_ship_by_id)` every tick with `sim_ship_lut_reset()` that clears only IDs inserted on the previous tick (‚â§ `ship_count` entries).

**Evidence:** Same pattern as `SnapShipLut` touched invalidation (2026-06-19). Typical idle server has 2 ships; full memset cleared 512 pointer slots unnecessarily.

**Risk:** None. `find_sim_ship` semantics unchanged.

---

### Change 4: O(1) claim-flag lookup in ship JSON builder

**File:** `server/src/net/websocket_server.c` ‚Äî `build_ships_blob_from_snapshot`

**What:** One pass over active claim flags builds `claim_by_ship_id[sid]`; per-ship lookup is O(1) instead of scanning up to 16 flags per sim ship.

**Evidence:** Up to 16 √ó 200 = 3200 comparisons per blob build when flags are active; idle server skips the build entirely when `claim_flag_count == 0`.

**Risk:** None. Output identical; guard preserved for zero flags.
