## Applied Changes

### Change 1: `copy_dropped_items_to_blob` — active-slot copy replaces full-array memcpy

**File:** `server/src/net/websocket_server.c`  
**Functions:** `copy_dropped_items_to_blob`, `blob_worker_submit_snapshot`, sync fallback

**What:** Replaced `memcpy(dropped_items, sizeof(dropped_items))` (~12 KB/tick) with field-by-field copy of active slots only (mirrors `copy_tombstones_to_blob`). Inactive slots are marked `active=false`; active slots are struct-copied. Tracks `dropped_item_active_count` on the snapshot.

**Evidence:** `DroppedItem` is ~48 B × 256 slots = 12 KB copied every tick even when 0 items are on the ground (typical idle state). Active-only copy moves 0 bytes when empty and scales with live drop count.

**Risk:** None. JSON builder already gates on `active`; snapshot semantics unchanged.

---

### Change 2: Fast-path empty `tombstones` / `droppedItems` JSON

**File:** `server/src/net/websocket_server.c` — `build_shared_blobs_from_snapshot`

**What:** When `tombstone_active_count == 0` or `dropped_item_active_count == 0`, emit `"[]"` directly (2 bytes) instead of scanning 64/256 slots on the blob-worker hot path. Counts are computed during snapshot copy (no extra global counter maintenance).

**Evidence:** Idle server has 0 tombstones and 0 dropped items every tick. Avoids 320 slot checks plus bracket/comma setup per 5 Hz idle tick (scales to 30 Hz under load).

**Risk:** None. Output identical to empty-loop result.

---

### Change 3: Skip redundant ground-NPC AOI world-position update

**File:** `server/src/net/websocket_server.c` — `build_shared_blobs_from_snapshot`

**What:** When `ship_id == 0` and the NPC entry is not dirty, reuse cached `npc_world_x/y` instead of reassigning from unchanged `x/y`. On-ship NPCs still recompute world position every tick (ship may move without NPC field changes).

**Evidence:** Mirrors the player ground-player optimization from 2026-06-22. With ~37 idle NPCs, saves two float writes per stationary ground NPC per tick.

**Risk:** None. Ground NPCs only skip reassignment when every serialized field (including `x/y`) is unchanged.

---

### Change 4: Bounded `claim_flags` snapshot copy + guarded ship JSON lookup

**File:** `server/src/net/websocket_server.c` — submit path, sync fallback, `build_ships_blob_from_snapshot`

**What:** Copy only `claim_flag_count` slots (high-water slot index, max 16) into the blob snapshot instead of the full array. Skip the per-ship claim-flag linear scan entirely when `claim_flag_count == 0`.

**Evidence:** Idle server has no active claim flags; avoids 16× ship_count comparisons per blob build and ~384 B redundant memcpy.

**Risk:** None. `claim_flag_count` is already maintained as slot high-water mark on plant/remove paths.

## Proposed (Needs Human Review)

### A. `per_gs_pool` -Warray-bounds: signed offset wrap

`per_gs_pool[100][524288]` is written via `_goff` (int). When `_goff` exceeds `PER_GS_BUF`, size arithmetic can wrap. Fix requires `size_t _goff` with a hard cap before every write. Deferred: correctness-critical send loop.

### B. `view_radius` hardcoded to 5000

Client-reported view radius is parsed but send path uses `const float _view_r = 5000.0f`. Changing this affects AOI fairness — gameplay decision.

### C. `placed_structure_count` linear scans in movement hot path

Multiple O(structures) scans per moving player for dock/shipyard lookup. Structure-ID hash would help but touches data layout.

### D. `handle_island_collisions` / `handle_ship_collisions`

Correctness-critical physics; no safe automatic change.

### E. `sim_update_spatial_hash` full 1.5 MB memset every tick

4096 cells × ~388 B cleared even when only a handful of entities exist. Incremental dirty-cell clearing needs careful collision correctness audit.

### F. UDP `snapshot.c` delta baseline O(n²) entity lookup

UDP snapshot path not active at runtime; deferred.
